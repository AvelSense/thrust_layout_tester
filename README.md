A utility script with 3D visualisation to test the output of arbitrary thrusters layout configurations.

## Installation
- Clone the repository and install the requirements (`pip install -r requirements.txt`)

## Usage
 - In `main.py`, modify the `motors` and `directions` variables to match the configuration you want to test.
 - Execute `main.py`
<img width="687" height="410" alt="configuration_variables" src="https://github.com/user-attachments/assets/ecb4ed9b-a3d3-4d62-a88b-16d624600cdc" />

 - The layout described by the variables will appear. The reference frame of the robot is indicated by the thinest lines (X red, Y green, Z blue), while the other lines represent each motor.
   They start at the position of the motor and follow its direction.
<img width="789" height="736" alt="visualisation_layout" src="https://github.com/user-attachments/assets/20082b91-313c-4462-b5bf-64759ab780ef" />

 - In the console, you can type the effort you want to be applied in the reference frame of the robot. Type e.g. `0 0 1 0 0 0` to test the ability of your robot to move vertically (along Z-axis).
 - The console will print if a solution was found (thrust generated by each thurster), and the actual output in the reference frame of the robot.
   Note that if the rank of the thrust matrix computed from your layout is <6, the actual output will differ from the target for at least some axes.
   The effort generated by each thrusters is showed in the 3D representation.
